  ---
  Problemas Remanescentes

  🟡 1. Fallback direto ao OpenAI continua bypassando o sistema de providers

  Arquivo: server/ai-cache.ts:199-223

  O console.warn foi adicionado (bom!), mas o comportamento continua: se nenhum provider estiver configurado no banco, o
   sistema usa OPENAI_API_KEY diretamente, ignorando a configuração da UI. Para o operador que configurou apenas
  Anthropic ou Gemini, o sistema silenciosamente tentaria usar OpenAI. O warn ajuda na depuração, mas a lógica ainda
  pode surpreender.

  ---
  🟡 2. secret: "" no session config em produção

  Arquivo: server/session-config.ts:57
  secret: sessionSecret || (!isProduction ? "dev-secret-inseguro" : ""),
  A string vazia "" como secret do express-session é tecnicamente inválida — a biblioteca aceita, mas assina os cookies
  com uma chave vazia, que é trivialmente forjável. O risco é baixo porque a linha 48-50 já lança erro antes de chegar
  aqui em produção, mas é um detalhe que pode causar comportamento inesperado se a validação for contornada algum dia.

  Sugestão: Substituir "" por uma string aleatória gerada em runtime (ex: crypto.randomBytes(32).toString('hex')) como
  último fallback de desenvolvimento, deixando claro que não é para produção.

  ---
  🟢 3. calculateNextRun — const no case weekly sem bloco

  Arquivo: server/routes/shared.ts:110-114
  case "weekly":
    const targetDay = dayOfWeek ?? 1;  // sem { }
    ...
    break;
  O caso monthly foi corretamente envolto em {} (linha 116), mas o weekly ainda declara const dentro do case sem bloco.
  Funciona em TypeScript/JS moderno, mas é inconsistente com o padrão adotado no caso monthly.
